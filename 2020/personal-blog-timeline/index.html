从零开始，建个个人博客

2020-04-10
技术
字数统计: 3k   |   阅读时长≈ 10 分钟

​ 经过两天时间的奋战，我终于建好了自己的blog；这个网站托管在GitHub上，可以基本确保安全；不过域名是有时间限制的。为了帮助未来的我自己或者别人，我觉得应该把建站的“一点人生的经验”记录下来，如果真的有人看的话，也算是“做了一点微小的贡献”。

[TOC]

一、从GitHub开始

  1. 首先注册一个Github账号，有的话可以忽略；

  2. 新建一个仓库（repositories），建议取名的时候注意下，比如你的GitHub用户名是username，给repositories取名时取名叫username.github.io。这样的好处是，以后直接在浏览器里访问username.github.io就可直达你的个人博客。不过前提是你的这个repositories里已经有网页文件了，通常是一个index.html文件，没有这个文件的话是打不开的；

  3. 为了验证是否能正常浏览，我们就要新建一个index.html文件。比较简单的方法是：打开Word，写几句话（不要搞什么花里胡哨的图片和形状，就是单纯的文字），然后保存为网页文件，并命名为index.html；

  4. 在你的仓库（repositories）下最左边找到 <>code，并upload你刚才做好的index.html；

  5. 在<>code所在的一行，最右边有个settings，里面有一个部分叫Github Pages，绿色背景的那一条里有你的网址，点击看看效果吧！

二、丰富你的网页

2.1 安装博客框架

​ 如果你是大佬的话，可以自己动手丰衣足食；如果是小白的话，可以像我一样，copy就完事了。

​ 网页也分为静态网页和动态网页。至于他们的不同我也不是很清楚，但是静态网页响应速度快，对于一个个人博客来说足够了：因为网页内容不多，更新也不会很频繁，功能基本仅限于展示，所以我选择了静态网页。

​ 制作静态网页的博客框架也有很多，我选择了Hexo。（Hexo官方文档链接：https://hexo.io/zh-cn/）

  1. 按照文档说明，在安装Hexo前要先安装Node.js（建议10.0以上版本）和Git；

  2. Git官网链接：https://git-scm.com/

  3. Node.js官网链接：https://nodejs.org/en/

  4. 安装Hexo需要打开console（快捷键Win+R，输入cmd），使用npm指令安装：

    npm install -g hexo-cli

  5. 在某处新建一个文件夹，并复制文件夹的地址；

  6. 执行命令：hexo init <folder address>

  7. 打开文件夹，按住shift和鼠标右键，在此处打开命令行窗口，执行：npm install

    至此博客框架就安装好了。

2.3 加一个模板

​ 在Hexo官方网页上，有一个Theme库，都是大家自己做的主题，可以选个自己喜欢的主题，应用到自己的网站上。具体的部署方法，主题的制作者应该都会写吧。我用的是Ayer，如果你喜欢这个网站主题，也可以用一样的主题；另外Hexo自带一个主题，也挺不错的。

​ 可以在现存模板的基础上加一些插件，比如音乐播放器，视频播放器等；需要的话可以自己去搜索。

2.4 查看效果

​ 有了心仪的主题，可以先看看效果了。

​ 在文件夹根目录启动Console，输入 hexo s，根据提示，打开浏览器，输入127.0.0.1:4000或者localhost:4000(实质上是启动服务器并输出到本机4000端口上)，即可预览效果。

​ 稍等片刻，在你的代码仓库里就能看到静态网页文件了，其中包括index.html。

​ 这时在浏览器里输入username.github.io就能看到你的网站了，理论上，不出意外的话。

2.6 一点人生的经验

​ 其实我在做的时候出了点问题，不知道是哪一步写错了代码，最后总是报错，所以实际上我是直接在theme作者源码上做的改进，还disable了很多功能。建议大家以后做好版本管理，多多backup。

三、自定义域名

​ 做到这里你已经很成功了，但是可以再锦上添花一下。username.github.io这样的网址我是不满意的，没有个性，所以我去买了个域名 ohmy-zsa.site。我个人觉得这个域名挺，可爱的，虽然我本意并非如此。因为我喜欢的女团 Oh My Girl的官方网站是 ohmy-girl.com,所以我才一时冲动买了这么个域名。

3.1 哪里买域名

​ 卖域名的地方很多，国内的比如阿里腾讯可能会需要备案（需要你的真实身份信息），为了避免喝茶，我选择在国外平台购买。我选择了godaddy，这个平台缺点是网页响应速度偏慢（中国移动的话可能会快点，因为服务器好像在香港，反正中国电信是慢成狗），优点是管理方面，自有DNS；最让我惊讶的是有微信小程序，更方便管理。

​ 一个域名通常几十块钱，取决于顶级域名是什么（指.com，.net这些）；而且通常第一年便宜，第二年开始就贵起来了，所以我都是只买一年，下一年再换个域名。

3.2 域名是啥

​ 似乎这一段应该是3.1，但是我懒得改。

​ 你在访问一个网页时，实质上是获取了一些文件，这些文件储存在GitHub服务器上。而网络设备要联网，就要有一个唯一的IP地址，比如IPv4地址 123.123.123.123；再细说其实是利用设备上的一个端口完成通信。而IP地址难以记忆，才有了域名，和域名解析服务。

​ 域名解析服务，即DNS，可将一个好记的网址连接到一个IP地址上，这样就方便了访问。

3.3 买好域名后

​ 其实3.2的内容不放到3.1也是有一定道理的，因为在godaddy上买域名要稍等一会才能使用。

​ 接下来再godaddy里找到域名的DNS设置。

​ 先ping一下username.gitub.io，可得到其IP地址。

​ 在DNS管理中删除非必需的记录，并添加两条解析记录：

类型 名称 内容 备注
A @（指本机） 一个IP地址
CNAME www username.github.io

​ 然后在代码仓库里的setting的GitHub Page里输入自己域名，稍等片刻，多刷新几次，等背景出现绿色即代表可使用这个域名；如果是蓝色则代表正在准备中。

​ 在变成绿色后可以看看是否能通过自定义域名访问网站。

3.4 一点人生的经验

​ 在此骂一下中国电信，实在是不厚道。

​ 如果这时候GitHub上是黄色背景并报错，说明出了问题~。

​ 在Console里使用ping和nslookup可查询域名指向的IP和DNS服务商，我发现正确设置后我的DNS被电信劫持了，都给我解析到长春市了，所以我无法通过自定义域名访问个人博客。

​ 解决方法是在设备中取消自动获取DNS，改成手动，并找一些公立的DNS比如 8.8.8.8和 114.114.114.114。

3.5 高级技巧

​ 如果通过上述方法可以正常访问，你会发现你用的是HTTP而非HTTPS，安全性总归差了那么一点，所以我想改成HTTPS。HTTPS是在HTTP的基础上应用TLS，通过TLS证书确保安全。这里的原理我也不是很懂，但我知道通过一些免费CDN可以解决，比如Cloudflare，因为它提供免费的TLS证书。

​ 在Cloudflare中选择托管域名，这样我的域名DNS就从godaddy到了Cloudflare——本来godaddy就提供了DNS服务器，现在要把它给Cloudflare的服务器。按照Cloudflare的提示，在godaddy里更换了Cloudflare提供的DNS服务器，然后需要等一段时间，可能十几分钟，更换成功后Cloudflare会给你发邮件的。

​ 同时Cloudflare会查找以前的解析记录，如果没有的话就重新添加一遍。

​ （存疑：可能需要先让橘色的云变成灰的，GitHub才能正常解析？还是说一直保持流量中转就可以？）

​ 这样再访问个人博客，就是完全的HTTPS加密了。

四、写作

4.1 创建文章

​ 可以访问博客、已经有了框架之后，我们就可以添加内容了。主目录输入 hexo new post [文章名]即可生成一个 [文章名.md]文件（在sourse/post里），我们就是要编辑这个.md文件。

4.2 怎么写文章

​ md文件是使用Markdown语法的文件，可以用记事本写，但是也有很多方便的编辑器，比如Atom和Typora，我个人只用过Typora。

​ Typora功能强大：可以方便地编辑标题、目录，插入图片、表格、代码段等；

​ Typora界面友好：没有多余干扰，字体漂亮，多种背景可选；

4.3 插入图片

​ 插入图片单独说，因为走了一些弯路。

​ 如果把图片放在网页里，意味着图片要从GitHub服务器送到我的设备上——本来GitHub在大陆速度就慢，这样图片传输就太耗时了，所以最好把图片储存在国内。这就需要一种叫图床的服务，本质上就是一个托管图片的服务器，这样加在图片就从国内直接加载，我个人用的是路过图床，当然也有其它一样好用的网站。

​ 上传图片到图床后，复制Markdown链接到编辑器Typora中，可直接显示出图片而不只是代码。

4.4 更新网页

​ 写好了文章，hexo s检查完，就可再次送到远端GitHub了。

​ 不过在检查前，一定要先执行 hexo clean或者手动删除根目录的public文件夹。

​ 然而这还不够，如果有的话，一定要删除.deploy_git文件夹，否则你预览和上传的都是旧版本网页。

​ dexo d一下就上传成功了。

​ 还有，上传后可能要到settings里检查一下，是否解析到正确的域名。

五、总结

​ 一口气写完这三千字，我太累了。

​ 以后有问题再更新吧。
